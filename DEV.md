# Гайд по розробці — Пекарня

## Як запустити

```
.\start-dev.bat
```

Відкриються два вікна консолі + браузер на http://localhost:5173

---

## Що за вікна і навіщо

```
┌─────────────────────────────────────────────────────┐
│  твій браузер  →  http://localhost:5173              │
│                        │                            │
│                   [Vite dev-сервер]                 │
│                  frontend/src/*.tsx                 │
│                        │                            │
│              API запити /api/... проксі             │
│                        ↓                            │
│               [FastAPI + uvicorn]                   │
│               backend/*.py                          │
│               database/bakery.db                    │
└─────────────────────────────────────────────────────┘
```

| Вікно | Процес | Порт | Навіщо |
|-------|--------|------|--------|
| Вікно 1 | Python / uvicorn | 8000 | бекенд: логіка, БД, API |
| Вікно 2 | Node / Vite | 5173 | фронтенд: UI з живим оновленням |

**Під час розробки завжди відкривай 5173.**

---

## Де що змінювати

```
backend/
  routers/      ← API ендпоінти (що повертає сервер)
  models/       ← таблиці бази даних (SQLAlchemy)
  schemas/      ← форми запитів/відповідей (Pydantic)
  services/     ← бізнес-логіка (розрахунки, правила)

frontend/src/
  pages/        ← сторінки (Замовлення, Випічка, ...)
  components/   ← спільні блоки (кнопки, таблиці, ...)
  api/          ← функції для запитів до бекенду

database/
  bakery.db     ← сама база (один файл SQLite)
  migrations/   ← зміни схеми (нові колонки, таблиці)
  schema.sql    ← повна схема з нуля
```

---

## Як працює "живе оновлення"

**Фронтенд (tsx/css):** збережи файл → браузер оновлюється сам за ~0.5 сек (HMR)

**Бекенд (py):** збережи файл → uvicorn перезапускається сам за ~1-2 сек (--reload)
→ У вікні з Python побачиш: `Reloading...`

**База даних:** якщо додав нову колонку до моделі — треба вручну зробити міграцію:
```python
# через Python або окремий скрипт
ALTER TABLE ... ADD COLUMN ...
```
і зберегти у `database/migrations/NNN_назва.sql`

## Коли і як робити коміт

```
1. Написав/виправив код
       ↓
2. Перевірив у браузері що все працює
       ↓
3. Скажи Клоду "зроби коміт" — він сам напише повідомлення і закомітить
       ↓
4. Код потрапляє на GitHub (git push)
```

**Реліз (оновлення продакшн-версії)** — тільки з твого дозволу, окремою командою.

---

## Продакшн vs Dev — різниця

| | Dev (розробка) | Продакшн (робота) |
|--|---|---|
| Запуск | `start-dev.bat` вручну | Task Scheduler автоматично при вході |
| Фронтенд | Vite на 5173 (живий) | Зібраний `frontend/dist` на 8000 |
| Перезавантаження | Автоматичне при зміні файлу | Лише через трей або перезапуск |
| Трей (іконка) | Не запускається | Запускається автоматично |
| Логи | У вікні консолі | `logs/bakery.log` |

## Частi питання

**Зміни у коді не видно в браузері?**
→ Перевір що відкритий 5173, а не 8000

**Помилка "no such column" у вікні бекенду?**
→ Модель має нову колонку якої ще нема в БД — потрібна міграція

**Порт 8000 зайнятий при запуску?**
→ Старий uvicorn не завершився — закрий вікно Python або перезапусти ПК

**Як перевірити API вручну?**
→ http://localhost:8000/api/docs — інтерактивна документація всіх ендпоінтів
