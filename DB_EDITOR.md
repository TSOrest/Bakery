# Редактор бази даних (DB Editor)

Вбудований інструмент для прямого перегляду і редагування SQLite-бази `bakery.db`.
Доступний лише для ролі **admin** через вкладку **🗄 БД** у головному меню.

---

## API ендпоінти

Всі ендпоінти вимагають Bearer-токен адміна.

| Метод | Шлях | Опис |
|-------|------|------|
| GET | `/api/v1/db-editor/tables` | Список таблиць з кількістю рядків |
| GET | `/api/v1/db-editor/tables/{table}/schema` | Схема: колонки, FK, індекси, DDL |
| GET | `/api/v1/db-editor/tables/{table}/data` | Дані з пагінацією (`?page=0&page_size=50`) |
| GET | `/api/v1/db-editor/tables/{table}/fk-options/{column}` | Опції для FK-поля (для dropdown) |
| PUT | `/api/v1/db-editor/tables/{table}/row/{pk_value}` | Оновити рядок за PK |
| DELETE | `/api/v1/db-editor/tables/{table}/row/{pk_value}` | Видалити рядок за PK |

## Як це працює

### Схема таблиці
Використовує SQLite PRAGMA:
- `PRAGMA table_info(table)` — колонки: назва, тип, NOT NULL, DEFAULT, PK
- `PRAGMA foreign_key_list(table)` — FK: звідки, куди, ON DELETE/UPDATE
- `PRAGMA index_list/index_info` — індекси та UNIQUE-обмеження
- `sqlite_master.sql` — DDL (CREATE TABLE)

### FK-поля (випадаючі списки)
При редагуванні рядка поля з FK автоматично замінюються на `<select>` з варіантами з referenced таблиці. Відображається "label (PK)" де label — перша знайдена колонка з `name`, `full_name`, `short_name`, `title`, `key`, `description`, `value`.

### Захист
- Лише роль `admin` має доступ (перевірка на рівні FastAPI через `get_current_user`)
- Назви таблиць і колонок передаються через URL, але валідуються проти `sqlite_master` перед виконанням
- Значення передаються як параметри (не конкатенація в SQL) — захист від SQL injection

---

## Обмеження

- Підтримуються тільки таблиці з одним PRIMARY KEY (що відповідає всім таблицям цього проекту)
- Не підтримується `INSERT` нових рядків — лише редагування і видалення
- Довгі значення (>60 символів) скорочуються в таблиці, але повністю видно при наведенні і в модалі редагування
- Пагінація: 50 рядків на сторінку

---

## Підтримка актуальності

При додаванні нових таблиць до схеми бази даних (`database/schema.sql` або міграції):
1. Додати опис таблиці до `TABLE_DESCRIPTIONS` у `DbEditorPage.tsx`
2. Якщо нова таблиця має нестандартну FK-колонку для відображення — перевірити `LABEL_CANDIDATES` у `db_editor.py`

> Редактор автоматично відображає всі таблиці через `sqlite_master`, але описи треба додавати вручну.
